# Agent Instructions

## Required Workflow

- After completing each new user requirement, commit and push the source changes
  to `main`.
- If the requirement changes the public frontend, rebuild the public frontend and
  push the generated deployment update to `gh-pages`.
- Before finishing a requirement, add a local entry to the requirement log below
  with the date, the new requirement, and the resulting commit when available.
- Do not commit credential contents. The GitHub token path is documented in
  `README.md` and `docs/public-frontend.md`; use it only through a non-echoing
  credential flow such as a temporary `GIT_ASKPASS` helper.
