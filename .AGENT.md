# Agent Instructions

## Required Workflow

- After completing each new user requirement, commit and push the source changes
  to `main`.
- If the requirement changes the public frontend, rebuild the public frontend and
  push the generated deployment update to `gh-pages`.
- Maintain `docs/code-structure.md` as the current project code-structure
  reference.
- Maintain `docs/functional-requirements.md` as the current project functional
  requirements reference.
- Whenever code changes, review and update both docs above in the same change
  so they remain synchronized with the implementation.
- Keep `.AGENT.md` focused on standing workflow rules. Do not use it as a
  per-change or per-requirement log.
- Do not commit credential contents. The GitHub token path is documented in
  `README.md` and `docs/public-frontend.md`; use it only through a non-echoing
  credential flow such as a temporary `GIT_ASKPASS` helper.
